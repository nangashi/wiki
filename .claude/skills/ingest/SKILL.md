# /ingest スキル

指定したソースを読み込み、wikiページを作成・更新する。

## 事前準備

1. `wiki/collections.md` を読み、コレクション構成と判定ルールを確認する
2. コレクション判定（後述）を実行してユーザーに確認を取る
3. 対象コレクションの `schema.md` を読み、ルールと共通書き込み手順を確認する

---

## コレクション判定

ソースの内容（URL・タイトル・概要）から、どのコレクションに追加するかを判定する。

`wiki/collections.md` の「判定目安」に照らして各コレクションへの該当可否を評価し、以下のフォーマットでユーザーに提示して確認を取る:

```
**コレクション判定**

- insight: [YES / NO / 候補] — 理由
- it: [YES / NO / 候補] — 理由

→ [判定コレクション] に追加しますか？
```

ユーザーが承認したコレクションに対してのみ以降の処理を実行する。複数コレクションへの追加も可。

---

## 引数フォーマットとソース種別

引数のパターンから自動的にソース種別を判定する。

| 引数の形式 | ソース種別 |
|-----------|-----------|
| `https://...` または `http://...` | Webページ |
| `.pdf` で終わるパス | PDF |
| `本:` または `book:` で始まる文字列 | 本に関するWeb収集 |
| `トピック:` または `topic:` で始まる文字列 | トピックに関するWeb収集 |

引数が複数ある場合は順に処理する。

---

## ソース種別ごとの処理

### Webページ

1. `WebFetch` で指定URLの内容を取得する
2. ページの主要な概念・情報を抽出する
3. **insightコレクションに取り込む場合**: ソースがSNS投稿・伝聞（三次ソース）のみなら、本質審査のS1基準によりページ作成不可となる。`WebSearch` で一次・二次ソース（原典・著者本人の文章・解説記事）を補足取得してから進める
4. sources frontmatter: `["<URL>"]`（補足取得したURLがあれば続けて列挙）

### PDF

1. `Read` ツールで指定PDFファイルを読み込む
2. ドキュメントの主要な概念・情報を抽出する
3. sources frontmatter: `["file:<ファイル名>"]`

### 本に関するWeb収集

書名（著者）を引数として受け取る。例: `本: リファクタリング（Martin Fowler）`

1. 以下の観点で `WebSearch` を複数回実行する（計5〜8回）:
   - 著者本人の文章（原著の抜粋・著者のブログや講演録・出版社の公式ページなど一次ソース）
   - 書籍の概要・目次
   - 主要概念・重要章のサマリ
   - レビュー・解説記事
   - 著者の他の著作との比較・関連
2. 収集したURLを一次（原典・著者本人）／二次（解説・書評）／三次（SNS・伝聞）に区分してリストアップする
   - **insightコレクションに取り込む場合**: 一次または二次を最低1つ確保する（本質審査のS1基準: 三次のみではページ作成不可）。確保できるまで検索を追加する
3. 取得した情報から主要概念を抽出する
4. sources frontmatter: `["book:書名（著者名）", "https://url1", "https://url2", ...]`
   - `book:` マーカーを先頭に置く

### トピックに関するWeb収集

トピック名を引数として受け取る。例: `トピック: 強化学習`

1. 以下の観点で `WebSearch` を複数回実行する（計5〜8回）:
   - トピックの定義・概要
   - 主要な技術・手法・概念
   - 代表的な論文・事例（原論文・公式ドキュメントなど一次ソースを優先）
   - 最近の動向・課題
2. 収集したURLを一次（原論文・原典・公式資料）／二次（解説記事）／三次（SNS・伝聞）に区分してリストアップする
   - **insightコレクションに取り込む場合**: 一次または二次を最低1つ確保する（本質審査のS1基準: 三次のみではページ作成不可）。確保できるまで検索を追加する
3. 取得した情報から主要概念を抽出する
4. sources frontmatter: `["topic:トピック名", "https://url1", "https://url2", ...]`
   - `topic:` マーカーを先頭に置く

---

## wikiへの書き込み

対象コレクションの `schema.md` の **「共通書き込み手順」** に従ってwikiページを作成・更新する。

複数コレクションに追加する場合は、コレクションごとに書き込み手順を独立して実行する。

---

## 出力形式

処理完了後、以下を報告する:

```
## 取込完了

**ソース**: [ソース種別] <入力値>

### [コレクション名] — 新規作成 (N件)
- [[スラグ]] — タイトル

### [コレクション名] — 更新 (N件)
- [[スラグ]] — タイトル（追加内容の要約）

### スキップ (N件)
- タイトル — 理由（例: 既に詳細に記録済み、採用基準に該当せず、一次・二次ソース未確保でS1保留）

### 追加したリンク
- [[A]] ↔ [[B]]: 関係の説明

### 本質審査（insightコレクションのみ）
- [[スラグ]] — 合格 / 改善（改善した基準と内容の要約）

### 補足調査未実施（該当時のみ）
- [[スラグ]] — 未実施の観点（ユーザーが補足調査を断った場合）

### Anki同期（insightを新規作成・更新した場合のみ）
- `/anki` で新規・変更ページを検知し、カードへの反映案を確認できます
```
