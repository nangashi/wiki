# /query スキル

wikiを検索して質問に答える。wikiに情報が不足している場合はWebで補完し、新たな知見があればwikiへの追加を提案する。

回答は **2段階パイプライン** で生成する: Sonnet（草稿）→ GPT-5.5 Critic（批評・構造化）。

## 事前準備

`wiki/collections.md` を読み、コレクション構成を確認する。

---

## 処理フロー

### ステップ1: wikiを検索する

1. `wiki/insight/index.md` と `wiki/it/index.md` を読み、全コレクションのページ一覧を把握する
2. 質問に関連しそうなページを特定する（スラグ名・index のサマリからあたりをつける）
3. 関連ページを `Read` で読み込む（コレクションをまたいで読む）
4. 関連ページからさらにリンクされているページも必要に応じて読む

### ステップ2: wiki情報で回答できるか判断する

**wikiで十分な場合**（以下をすべて満たす）:
- 質問の核心部分をwikiが直接カバーしている
- 情報が具体的で、あいまいな記述でない
- 追加の文脈なしに理解できる

→ **ステップ3（Wiki回答）** へ

**Webを使う場合**（いずれかに該当）:
- wikiに該当概念のページがない
- wikiにページはあるが情報が不足・断片的
- 最新情報が必要な質問

→ **ステップ4（Web補完）** へ

### ステップ3: wikiから回答する

wikiの情報をもとに回答を構成する。引用元のページを `[[スラグ]]` で明示する（クロスコレクション参照の場合は `[[insight:スラグ]]` / `[[it:スラグ]]` 形式）。

その後、**ステップ5（Critic実行）** へ。

### ステップ4: Webで情報を補完する

1. 質問に関連するキーワードで `WebSearch` を実行する（1〜3回）
2. 得られた情報を統合して回答する
3. 参照したURLをリストアップする

その後、**ステップ5（Critic実行）** へ。

### ステップ5: GPT-5.5 Critic を実行する

ステップ3または4で生成した草稿を GPT-5.5 に渡し、批評・構造化を依頼する。

Agent ツールで以下のサブエージェントを **バックグラウンドで** 起動する:

```
subagent_type: codex:codex-rescue
run_in_background: true
prompt: 以下のプロンプトテンプレートを使用（草稿・質問・wiki情報を埋め込む）
```

**Critic プロンプトテンプレート**:

```
あなたはwikiクエリのCriticエージェント（GPT-5.5）です。
作業ディレクトリ: /home/toyama-ryosuke/ghq/github.com/nangashi/wiki

## 元の質問
{question}

## Sonnet 4.6による草稿レポート
{draft}

## 指示
1. wikiを調べる（bashコマンドで）:
   - cat /home/toyama-ryosuke/ghq/github.com/nangashi/wiki/wiki/insight/index.md
   - 草稿が参照するwikiページを確認し、主張の根拠を検証する
   - 草稿が見落としているwikiページを探す

2. 以下の観点で草稿を批評・補完し、最終レポートを出力する:
   - wikiに根拠のない主張を除外または「wiki外の知見」として明示する
   - 草稿の並列列挙を因果連鎖・階層構造に再構成する
   - 草稿が見落とした上流の問い・視点を追加する
   - 草稿の各節が「なぜ」を説明しているか確認し、不十分なら補強する

3. 出力形式:
   ## 最終回答
   <構造化・補強された回答>

   ## 草稿への批評（差分）
   - 追加した視点: ...
   - 除外・修正した主張: ...
   - 構造の変更点: ...

   ## 参照
   - wiki: [[スラグ]] ...
```

Critic の完了を待ち、結果を受け取ったら**ステップ6**へ。

### ステップ6: 最終出力を構成する

草稿（ステップ3/4）と Critic の出力（ステップ5）を統合して最終回答を出力する。

その後、**ステップ7（wiki追加判断）** へ。

### ステップ7: wikiへの追加を判断する

**以下のいずれかに該当する場合のみ** wiki追加を提案する:

| 追加が必要な状況 | 説明 |
|----------------|------|
| **新たな知見** | wikiに存在しない情報・概念・事実が得られた |
| **ページ間の新接続** | 既存の2つ以上のwikiページが実は関連していることが判明した（リンク未設定） |
| **既存ページの補強** | 既存ページの説明に重要な補足情報が加わる |

追加を提案する場合、`wiki/collections.md` の判定目安に照らしてどのコレクションに追加するかも提示する。

**追加不要な場合** (提案しない):
- 単純にwikiの内容を言い換えただけ
- 質問への直接回答だが、wiki的な知識としての独立性がない（例: 「〇〇の設定コマンドは？」）
- 既にwikiに記録済みの内容

### ステップ8: wiki追加を実行する（提案が承認された場合）

ユーザーが追加に同意した場合、対象コレクションの `schema.md` の **「共通書き込み手順」** に従ってwikiページを作成・更新する。

---

## 出力形式

```
## 回答

<Criticによる構造化・補強済みの回答本文>

参照: [[スラグ1]], [[insight:スラグ2]]（wikiから回答した場合）
参照: https://url （Webから補完した場合）

---

## Criticによる差分メモ（参考）

- 追加した視点: ...
- 除外・修正した主張: ...
- 構造の変更点: ...

---

## wiki追加の提案（該当する場合のみ）

以下の知見をwikiに追加することを提案します:

### 提案1: [新規ページ or 追記] — タイトル
**コレクション**: insight / it
**理由**: 新たな知見 / ページ間の新接続 / 既存ページの補強
**内容**: <追加する情報の要約>
**対象**: [[スラグ]]（追記の場合）

---

追加しますか？（「はい」で実行、「いいえ」でスキップ）
```

---

## 注意事項

- 回答の質を落としてでもwikiに情報を詰め込もうとしない
- Webの最新情報とwikiの情報が矛盾する場合は、その矛盾をユーザーに伝える（`/lint` の実行も提案する）
- Criticステップは回答の深さと構造を高めるためのもの。Criticが草稿を大幅に変更した場合、その理由をユーザーに簡潔に説明する
