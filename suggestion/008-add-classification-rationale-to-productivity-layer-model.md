# 提案: [[productivity-layer-model]] に分類軸の根拠を追加する

## 対象

- `wiki/insight/pages/productivity-layer-model.md`

## 判定

不合格寄り。分類・タクソノミー型だが、6層で切る必然性が弱い。

## 根拠

概要は「生産性を6つの層に分解し、層間には自動変換が起きない抵抗がある」と述べている。これは分類として有用だが、`schema.md` と `essence-review-criteria.md` の C6 は「なぜこの分類軸が有効か」を必須としている。

現状では「6層がある」ことと「層間抵抗がある」ことは示されているが、なぜこの6層でなければ AI ツール導入の波及不全を説明できないのかが弱い。

## 改善案

概要または詳細冒頭に、分類軸の正当化を追加する。

例:

> この6層で切る理由は、生産性改善が「個人の処理能力」「チームの協調」「組織の制度」「事業成果」の間で評価主体と制約条件を変えるためであり、隣接層を同一視すると改善がどこで失速したかを特定できなくなるからである。

あわせて、各層間で変換が止まる典型原因を1行ずつ書くと C2 も強くなる。
