# 提案: [[problem-driven-selection]] の核心を「原理」に引き上げる

## 対象

- `wiki/insight/pages/problem-driven-selection.md`

## 判定

不合格寄り。概要が「何をするべきか」の原則で止まり、「なぜそうなるか」の説明が弱い。

## 根拠

概要は「技術・手法・ツールを選ぶ際に、何ができるかではなく何を解くかを評価軸の先頭に置く」と述べる。これは実践方針としては正しいが、`essence-review-criteria.md` の C1 は「このページは X という原理を主張している」と即座に言えることを求めている。

現状の核心は「問題を先に置け」という規範に見え、問題不在の選択がなぜ失敗するかの生成メカニズムが薄い。

## 改善案

概要冒頭を、次のように因果原理へ寄せる。

> 選択の良し悪しは選択肢の性能ではなく、解くべき制約との適合で決まるため、問題が定義されていない状態では「より高機能な選択肢」ほど評価軸を乗っ取りやすい。

そのうえで、詳細の各項目を「機能評価が問題定義を代替してしまう」「比較軸が増えるほど選択が目的から離れる」といった C2 の接続で書き直す。
